# 🛡️ Game Economy Monitor (遊戲經濟防禦監控系統) - PoC

![Status](https://img.shields.io/badge/Status-PoC_Completed-success.svg)

這是一個以 Node.js 開發的自動化市場價格監控腳本 (Proof of Concept)，專為線上遊戲（如《新楓之谷》）的營運風控團隊設計。透過自動化巡邏外部第三方交易平台（8591），即時捕捉異常的虛擬資產傾銷行為，並透過 Discord Webhook 發送警報，實現「攻防一體」的數據驅動營運策略。

## 🎯 專案動機與痛點分析 (Pain Points)

在 MMORPG 的生態系中，遊戲幣與高價值道具的穩定性是維持產品生命週期的核心。然而，傳統營運團隊常面臨以下挑戰：
1. **黑產傾銷干擾**：外掛工作室或代練團隊大量產出資源並低價拋售，嚴重破壞遊戲內經濟體系與玩家成長體驗。
2. **人工巡查耗時**：依賴營運人員手動定期巡視外部交易網，不僅耗費大量人力成本，且存在嚴重的時間差。
3. **防禦反應過慢**：當官方發現經濟異常時，往往大量黑幣已流入市面，難以進行有效的源頭阻斷。

## 💡 解決方案 (The Solution)

本專案將複雜的營運痛點轉化為自動化的量化方案。透過建立即時的市場監測與異常行為預警系統，取代傳統人工盯盤。

### ✨ 核心功能
* **🕒 定時自動化巡查**：利用 `node-cron` 建立背景排程，全天候每 10 分鐘自動抓取指定交易網頁的最新商品資訊。
* **🔍 精準數據解析**：透過 `cheerio` 進行 DOM 節點解析，精準鎖定並過濾出目標商品的「標題」與「真實價格」。
* **⚠️ 閾值觸發機制**：內建價格防線邏輯，當市場最低價跌破設定的健康閾值（例如：幣值異常貶值）時，即判定為潛在的黑產傾銷行為。
* **🚨 跨平台即時警報**：整合 Discord Webhook，異常發生當下秒級推播警報至營運團隊群組，大幅縮短發現時間 (MTTD)。

## 🛠️ 技術棧 (Tech Stack)

* **Runtime**: Node.js
* **HTTP Client**: `axios` (網頁原始碼請求與 Webhook 發送)
* **HTML Parser**: `cheerio` (解析第三方平台網頁結構與精準擷取價格標籤)
* **Task Scheduler**: `node-cron` (實現背景定期執行邏輯)
* **Environment Management**: `dotenv` (保護 Webhook URL 等敏感資訊)

## 📈 商業價值與影響力 (Business Impact)

1. **極大化營運回報**：協助公司回收外部代練與黑產流失的利益，確保遊戲生態平衡。
2. **決策科學化**：將「主觀感覺市場怪怪的」轉化為「客觀的即時數據警報」，為後續的防外掛/防代練機制提供數據佐證。
3. **流程效率優化**：釋放營運團隊的勞動力，將人力資源集中於更高價值的產品企劃與玩家體驗優化上。

## 🚀 未來架構拓展 (Future Roadmap)

目前版本為輕量級的 PoC 驗證，未來可針對實際業務場景進行以下升級：
1. **容器化部署 (Docker)**：將本服務打包為 Docker Image，部署至雲端伺服器 (如 AWS/GCP)，確保 24/7 高可用性與快速擴展。
2. **資料庫沉澱與視覺化**：導入 MySQL 或 MongoDB 記錄歷史價格波動，並串接 Python (Pandas) 進行長期數據清洗與趨勢視覺化分析。
3. **AI 視覺監控整合**：對於無法單純依賴網頁文本的防禦場景，規劃結合 YOLO 物件偵測技術，開發終端遊戲畫面的自動化違規行為巡查工具。
